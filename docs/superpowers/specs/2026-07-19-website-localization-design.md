# 官网中英文本地化设计

**日期：** 2026-07-19
**范围：** `product-website` 独立官网

## 目标

官网仅支持英文与简体中文。首次访问显示英文；用户可在页头切换语言，所选语言保存在当前浏览器的本地存储中，并在后续访问时恢复。

## 方案

采用轻量的页面级本地化层，不引入额外国际化依赖：

1. 使用 `en` 与 `zh-CN` 作为受限语言标识；默认值为 `en`。
2. 将导航、英雄区、功能卡片、场景、隐私承诺、页脚及辅助文本集中为双语内容对象；React 组件只读取当前语言内容。
3. 封装语言偏好读取与写入：读取 `localStorage` 的合法值，缺失或无效时回退到英文；用户切换后立即保存。
4. 页头增加可访问的 `EN / 中文` 语言切换控件，切换即时更新页面，并同步设置根元素的 `lang` 属性（`en` 或 `zh-CN`）。

## 边界与兼容性

- 仅改动官网，不涉及浏览器扩展的既有国际化或运行环境。
- 不按浏览器系统语言自动选择，以保证首次访问始终为英文。
- 本地存储不可用时仍可正常展示英文并切换当前会话语言。

## 验证

新增与更新 Vitest 测试，覆盖：

- 双语内容的完整性与导航锚点一致性；
- 默认语言为英文；
- 合法的已保存偏好被恢复，无效值回退英文；
- 切换语言会更新状态并写入本地存储。

最后执行 `npm test` 与 `npm run build`，确保测试、类型检查和站点构建均通过。
